## 🧭 使用方式

- **换机器 / 重装**：`git clone git@github.com:gengshenchen/aimds.git`，打开对应 md 照做。
- **交给 AI 执行**：把文档链接或内容发给 AI，说「按这个手册配置，我的环境是 ___」。
- **凭据类**：文档里的密钥/UUID/密码等都是**示例**，实际操作按文档说明重新生成，切勿直接复用。

## ✍️ 写作约定（新增文档时遵守）

1. **文件名**：小写短横线，`主题-类型.md`（如 `proxy-runbook.md`、`k8s-notes.md`）。
2. **自包含**：每篇开头写清「用途 / 前提 / 目标」，正文给**可直接执行的命令**（带 `<占位符>`），别只写结论。
3. **带验证**：关键步骤后给一条「怎么确认成功」的命令或判据。
4. **排错速查表**：把踩过的坑整理成「症状 → 原因 → 解法」表格放文末——这是复用价值最高的部分。
5. 新增后**回到本 README 的索引表加一行**。
